### **Comparison of Quantum Network Simulators**
| **Simulator** | **Main Features** | **Pros** | **Shortcomings** |
|-------------|------------------|-----------|----------------|
| **SimulaQron** | - Simulates quantum internet applications in a platform-independent way.<br> - Assumes a network of quantum processors interconnected by quantum channels.<br> - Provides a universal interface (CQC - Classical Quantum Combiner) for integrating classical and quantum networking. | ✔ Platform-independent.<br>✔ Provides a universal interface for quantum applications.<br>✔ Integrates classical and quantum programming practices.<br>✔ Easy to use for application-layer testing. | ❌ Does not simulate physical-layer quantum effects.<br>❌ Lacks detailed quantum noise modeling.<br>❌ Not designed for low-level hardware simulation. |
| **QuNetSim** | - Focuses on quantum networking protocols.<br> - Provides synchronization for qubit transmission.<br> - Uses backend quantum simulators (SimulaQron, ProjectQ, QuTiP, EQSN). | ✔ Real-time simulation capability.<br>✔ Easy integration with backend quantum simulators.<br>✔ High-level network-layer simulation. | ❌ No built-in quantum error modeling.<br>❌ Limited realism in physical-layer quantum operations.<br>❌ Questionable feasibility of real-time interaction with hardware. |
| **SQUANCH** | - Parallelized quantum networking simulation.<br> - Supports distributed quantum processing.<br> - Simulates classical and quantum channels with error models.<br> - Tracks entanglement fidelity and synchronization. | ✔ Multi-agent parallel simulation.<br>✔ Includes error modeling for quantum channels.<br>✔ Tracks entanglement fidelity over time.<br>✔ Good for simulating distributed quantum networks. | ❌ High-level abstraction may lack accuracy in real-world quantum systems.<br>❌ No built-in quantum memory modeling.<br>❌ Limited flexibility for real-time applications. |
| **QuISP** | - OMNeT++ based simulator.<br> - Efficiently tracks quantum state errors.<br> - Introduces ruleset-based operation similar to SDN flow tables.<br> - Models physical and network-layer quantum communication. | ✔ Scalable due to focus on error tracking rather than full quantum state tracking.<br>✔ Integrates ruleset-based operations for efficient entanglement management.<br>✔ Models noise and errors effectively.<br>✔ Well-suited for network layer studies. | ❌ Does not track detailed quantum states (focuses on errors only).<br>❌ Less useful for quantum computing simulations.<br>❌ Limited support for heterogeneous quantum network architectures. |
| **SeQUeNCE** | - High-precision quantum network simulator.<br> - Models quantum state encoding in time bins, polarization, and quantum memories.<br> - Supports discrete event-based simulation with picosecond resolution.<br> - Modularized design with five key modules. | ✔ Realistic quantum network simulation with fine-grained time modeling.<br>✔ Supports multiple quantum state encoding methods.<br>✔ High modularity for flexible network design.<br>✔ Good for fidelity and timing-sensitive quantum applications. | ❌ High computational cost due to detailed modeling.<br>❌ Not optimized for large-scale quantum networks.<br>❌ Complex to configure for non-experts. |
| **NetSquid** | - Python-based discrete event quantum simulator.<br> - Models all major quantum network components.<br> - Provides detailed quantum state tracking with various representations.<br> - Supports modular network simulation. | ✔ High modularity and flexibility.<br>✔ Supports multiple quantum state representations (density matrix, stabilizer states, etc.).<br>✔ Suitable for protocol development and quantum network architecture design.<br>✔ Well-structured API for advanced users. | ❌ Computationally intensive for large-scale simulations.<br>❌ Complex for beginners.<br>❌ Requires Python expertise to use effectively. |
| **QKDNetSim** | - Simulates classical traffic in a QKD network.<br> - Implements ETSI QKD standards.<br> - Uses NS-3 network simulation framework.<br> - Focuses on QKD-specific network protocols. | ✔ Follows ETSI QKD standards for secure key distribution.<br>✔ Can be used for testing real-world QKD network implementations.<br>✔ Good for analyzing classical traffic impact on QKD. | ❌ Limited to QKD networks, does not simulate general quantum networks.<br>❌ Does not model quantum entanglement operations beyond QKD.<br>❌ Lacks flexibility for broader quantum networking research. |
| **QuNET** | - Quantum entanglement network simulator in Julia.<br> - Implements cost-vector analysis for entanglement routing.<br> - Supports both ground and space-based quantum networks.<br> - Uses temporal meta-graphs to simulate quantum memories. | ✔ Highly efficient cost-vector analysis.<br>✔ Scalable for complex quantum networks.<br>✔ Supports both terrestrial and satellite quantum networks.<br>✔ Can be used for real-time routing optimization. | ❌ Highly abstracted, lacks low-level quantum physics modeling.<br>❌ Does not provide detailed quantum error analysis.<br>❌ Limited to entanglement routing applications. |
| **NetQASM** | - Low-level instruction set architecture for quantum networks.<br> - Extends QASM for quantum communication.<br> - Provides platform-independent quantum programming interface.<br> - Integrates with SimulaQron and NetSquid. | ✔ Universal and extendable quantum instruction set.<br>✔ Enables high-level programming for quantum internet applications.<br>✔ Works across different quantum network architectures.<br>✔ Supports hybrid classical-quantum operations. | ❌ Not a network simulator per se, but rather an instruction set.<br>❌ Requires compatible quantum network hardware/software to be useful.<br>❌ Limited for standalone quantum network research. |
